# Chapter 1: Programming Paradigms

## 1.1 Introduction

A **programming paradigm** is a fundamental style or approach to programming that dictates how a developer structures, organizes, and writes code to solve problems. It defines the methodology of thinking about and implementing software solutions. Most modern languages (including Python) are **multi-paradigm**, supporting more than one style.

## 1.2 Different Programming Paradigms

> 📝 **Explain different programming paradigm with example for each. [4 marks] (2081 Ashwin - IOE)**

### Imperative Programming

Focuses on **how** to perform a task — the programmer writes explicit step-by-step instructions that change the program's state using variables, loops, and conditionals.

```c
// C example: sum of 1 to 5
int sum = 0;
for (int i = 1; i <= 5; i++) {
    sum += i;
}
```

### Procedural Programming

A subset of imperative programming that organizes code into reusable **procedures (functions)**. It follows a top-down, sequential approach.

```c
// C example
int add(int a, int b) { return a + b; }
int main() {
    printf("%d", add(3, 4));
}
```

### Object-Oriented Programming (OOP)

Organizes code around **objects** — instances of classes that bundle data (attributes) and behavior (methods) together. Built on four pillars: encapsulation, abstraction, inheritance, and polymorphism.

```python
class Dog:
    def __init__(self, name):
        self.name = name
    def bark(self):
        print(f"{self.name} says Woof!")

d = Dog("Buddy")
d.bark()
```

### Functional Programming

Treats computation as the evaluation of **pure mathematical functions**. Emphasizes immutability, no side effects, and higher-order functions. Programs are built by composing functions.

```python
from functools import reduce
nums = [1, 2, 3, 4, 5]
total = reduce(lambda a, b: a + b, nums)  # 15
```

### Declarative Programming

Focuses on **what** the result should be, not how to achieve it. The programmer describes the desired outcome and the language/runtime figures out the execution. SQL and HTML are common examples.

```sql
SELECT name FROM students WHERE marks > 80;
```

## 1.3 Advantages, Disadvantages of Different Paradigms

> 📝 **Discuss about the different programming paradigms with their advantage and disadvantage. [4 marks] (2082 Baishakh - IOE)**

### Imperative / Procedural

- **Advantages:** Simple to understand and trace; maps closely to hardware; efficient execution; fine-grained control over state and memory.
- **Disadvantages:** Code can become complex and hard to maintain as size grows ("spaghetti code"); mutable state increases risk of bugs; difficult to parallelize.

### Object-Oriented

- **Advantages:** Excellent modularity and code reuse through inheritance and composition; easier to model real-world entities; encapsulation provides data security; scalable for large projects.
- **Disadvantages:** Requires more upfront design and boilerplate; deep inheritance hierarchies can become complex; slightly slower due to object creation overhead; overkill for small, simple programs.

### Functional

- **Advantages:** Easier to reason about due to immutability and no side effects; naturally suited for parallel/concurrent processing; concise and expressive code; easier to test and debug.
- **Disadvantages:** Steep learning curve; can be less intuitive for programmers used to imperative style; higher memory consumption due to immutable data structures; recursion-heavy code can be less efficient.

### Declarative

- **Advantages:** Highly readable and concise; abstracts away implementation details; less error-prone for domain-specific tasks.
- **Disadvantages:** Less control over execution; can be slower due to abstraction overhead; limited to specific domains (SQL for databases, HTML for markup).

**Examples of languages by paradigm:**

- Imperative/Procedural: C, Pascal, FORTRAN
- Object-Oriented: Java, C++, Python, C#
- Functional: Haskell, Lisp, Erlang, Scala
- Declarative: SQL, HTML, Prolog
- Multi-paradigm: Python, JavaScript, Scala, C++

# Chapter 2: Introduction to Python Programming

## 2.1 Need of Python

Python addresses several shortcomings of older languages:

- **Rapid Development:** Python's concise syntax allows developers to write programs much faster than in C/C++ or Java. A task requiring 20 lines in Java may take 5 lines in Python.
- **Readability:** Python enforces clean indentation-based structure, making code inherently readable and maintainable.
- **Versatility:** A single language usable for web development, data science, AI/ML, automation, scripting, scientific computing, and more — reducing the need to learn multiple specialized languages.
- **Beginner-Friendly:** Low barrier to entry makes it ideal for teaching programming and for non-programmers (scientists, analysts) who need to code.
- **Rich Ecosystem:** Massive collection of third-party libraries and frameworks (NumPy, Django, TensorFlow, Pandas) means developers rarely need to build from scratch.
- **Community Support:** One of the largest and most active programming communities, ensuring abundant documentation, tutorials, and help.

## 2.2 History

- **Late 1980s:** Guido van Rossum began designing Python at Centrum Wiskunde & Informatica (CWI), Netherlands, as a successor to the ABC language.
- **1991:** Python 0.9.0 released — included classes, exception handling, functions, and core data types (list, dict, str).
- **1994:** Python 1.0 released — added lambda, map, filter, and reduce.
- **2000:** Python 2.0 released — introduced list comprehensions, garbage collection (reference counting + cycle detection), and Unicode support.
- **2008:** Python 3.0 released — a major backward-incompatible redesign to fix fundamental flaws (e.g., `print` became a function, integer division changed, all strings became Unicode by default).
- **2020:** Python 2 reached end-of-life (January 1, 2020). Python 3 is now the only supported version.
- **Named after** the British comedy group "Monty Python's Flying Circus," not the snake.

## 2.3 Features and Limitations of Python

> 📝 **List out the features and limitations of python. [2+2 marks] (2081 Ashwin - IOE)**

### Features

1. **Simple and Easy to Learn:** English-like syntax with minimal boilerplate. Uses indentation instead of braces for blocks.
2. **Interpreted Language:** Code is executed line by line by the Python interpreter, enabling interactive testing and rapid prototyping. No separate compilation step needed.
3. **High-Level Language:** Abstracts away low-level details like memory management, hardware interaction, etc.
4. **Dynamically Typed:** Variable types are determined at runtime — no need for explicit type declarations (`x = 10` automatically makes `x` an integer).
5. **Multi-Paradigm:** Supports procedural, object-oriented, and functional programming styles within a single language.
6. **Platform Independent (Portable):** Python code runs on Windows, Linux, macOS, etc., without modification. "Write once, run anywhere."
7. **Extensive Standard Library:** "Batteries included" philosophy — modules for file I/O, regex, networking, HTTP, JSON, math, OS interaction, etc., come built-in.
8. **Open Source and Free:** Freely available under the PSF license. Anyone can download, use, modify, and distribute it.
9. **Embeddable and Extensible:** Python can be embedded within C/C++ programs, and C/C++ code can be called from Python for performance-critical tasks.
10. **Automatic Memory Management:** Uses garbage collection (reference counting + cyclic garbage collector) to handle memory automatically.
11. **Large Community and Ecosystem:** Thousands of third-party packages available via PyPI (pip).

### Limitations

1. **Slower Execution Speed:** Being interpreted and dynamically typed, Python is significantly slower than compiled languages (C, C++, Java) for computation-heavy tasks.
2. **Not Ideal for Mobile Development:** Python is not natively used for mobile app development (Android/iOS). Frameworks like Kivy exist but are not mainstream.
3. **High Memory Consumption:** Python objects carry overhead, leading to higher memory usage compared to C/C++.
4. **Global Interpreter Lock (GIL):** The GIL in CPython prevents true parallel execution of multiple threads on multi-core processors, limiting CPU-bound multi-threaded performance.
5. **Runtime Errors:** Dynamic typing means type-related errors are caught only at runtime, not at compile time.
6. **Database Access:** Python's database access layer is less mature and less performant compared to JDBC (Java) or ADO.NET (C#).
7. **Not Suitable for Low-Level Programming:** Cannot be used for OS kernels, device drivers, or embedded systems requiring direct hardware control (unlike C/C++).

## 2.5 Top Python Implementations

1. **CPython:** The default and official implementation, written in C. Compiles Python code to bytecode and executes it on the CPython virtual machine. Supports C-extensions. Has the GIL.

2. **PyPy:** An alternative implementation focused on speed. Uses a Just-In-Time (JIT) compiler that translates Python bytecode to native machine code at runtime, making it significantly faster than CPython for long-running programs. Written in RPython.

3. **Jython:** Python implementation running on the Java Virtual Machine (JVM), written in Java. Allows seamless use of Java classes and libraries from Python code. Useful for Java-Python integration.

4. **IronPython:** Python implementation for the .NET framework, written in C#. Allows integration with .NET libraries and languages (C#, VB.NET). Does not have a GIL, enabling true multi-threading on .NET.

5. **MicroPython:** A lean, efficient implementation of Python 3 optimized for microcontrollers and embedded systems (e.g., ESP32, Raspberry Pi Pico). Includes a subset of the standard library tailored for hardware interaction (GPIO, sensors).

6. **Cython:** Not a pure Python implementation but a programming language that is a superset of Python. It allows writing C extensions for Python, compiling Python-like code to C for significant speed improvements. Used heavily in scientific libraries.

7. **Stackless Python:** A modified version of CPython that avoids using the C call stack, enabling micro-threads (tasklets) for massive concurrency without OS thread overhead.
